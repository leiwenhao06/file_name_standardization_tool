# 实验报告文件名批量规范化工具

一个专为学习委员设计的 Python 命令行工具，用于快速将同学们提交的实验报告（Word 文档）按照预设规范自动重命名，支持多科目、多实验次数，并可根据班级花名册自动生成配置文件。

## 功能特性

- ✅ **智能匹配**：根据文件名中的关键词（通常是同学姓名）自动识别报告所属者
- ✅ **多科目支持**：可针对不同科目维护独立的命名规范映射表
- ✅ **实验次数动态替换**：通过命令行参数指定实验/实践次数，自动生成如 `实验1_学号_姓名` 的标准名称
- ✅ **预览模式**：默认仅显示重命名预览，确认无误后再执行实际修改，防止误操作
- ✅ **冲突检测**：自动检查目标文件名是否已存在，避免覆盖
- ✅ **配置文件自动生成**：提供独立脚本，可根据班级花名册 Excel 一键生成所有科目的映射配置
- ✅ **扩展性强**：支持自定义正则表达式匹配、支持多种文件格式（默认 `.doc` / `.docx`）
- ✅ **未交人员名单**：处理完成后自动输出未提交报告的同学名单，方便催交
- ✅ **纯 Python 实现**：无需安装第三方库（除自动生成配置可选 `pandas`），兼容性好

## 快速开始

1. **配置映射表**  
   编辑 `config.py` 文件，或使用提供的脚本根据花名册自动生成（见下文）。

2. **运行预览**  
   ```bash
   python main.py /path/to/reports -s 操作系统 -e 1
   ```

3. **执行重命名**  
   ```bash
   python main.py /path/to/reports -s 操作系统 -e 2 --execute
   ```

## 详细配置说明

配置文件 `config.py` 是工具的核心，它定义了两个部分：
- **科目映射字典**：每个科目一个字典，键为用于匹配文件名的关键词（通常为姓名），值为标准文件名模板（可包含 `{exp_num}` 占位符）。
- **科目索引字典 `SUBJECTS`**：将科目名称映射到对应的字典，供命令行参数 `-s` 选择。

### 从花名册 Excel 自动生成配置

项目中提供了 `generate_config.py` 脚本，可根据班级花名册自动生成 `config.py`。

**花名册格式要求**：
- Excel 文件（`.xlsx` 格式）
- 第一列：学号
- 第二列：姓名
- 无标题行

示例（`班级花名册.xlsx`）：
| 学号    | 姓名 |
|---------|------|
| 2026001 | 张三 |
| 2026002 | 李四 |
| ...     | ...  |

**运行生成脚本**：
```bash
python generate_config.py "你的班级花名册文件路径.xlsx"
```
执行后将在当前目录生成 `config.py` 文件，内含预设科目（操作系统、计算机组成原理）的完整映射字典。

**自定义科目模板**：  
如需修改科目名称或命名格式，可编辑 `generate_config.py` 中的 `SUBJECT_TEMPLATES` 列表：
```python
SUBJECT_TEMPLATES = [
    ("操作系统", "实验{exp_num}_{学号}_{姓名}"),
    ("计算机组成原理", "实验{exp_num} - {学号} - {姓名}"),
    # 在此添加新科目
]
```

### 手动编写配置文件

如果不想使用自动生成，也可以手动编写 `config.py`，格式如下：

```python
# config.py
OS_MAPPING = {
    "张三": "实验{exp_num}_2024001_张三",
    "李四": "实验{exp_num}_2024002_李四",
}

SUBJECTS = {
    "操作系统": OS_MAPPING,
    # 添加更多科目...
}
```

> **提示**：键（姓名）将作为正则表达式进行匹配，默认会进行转义（普通字符串匹配）。如需使用正则表达式，请参考 [高级功能](#高级功能) 章节。

## 使用方法

### 命令行参数

| 参数 | 简写 | 是否必填 | 说明 |
|------|------|----------|------|
| `directory` | （位置参数） | 否 | 要处理的文件夹路径，默认为当前目录 `.` |
| `--subject` | `-s` | **是** | 科目名称，必须与 `config.py` 中 `SUBJECTS` 的键完全一致 |
| `--exp-num` | `-e` | **是** | 实验/实践次数，用于替换映射值中的 `{exp_num}` 占位符 |
| `--execute` | | 否 | 加上此参数后实际执行重命名；不加则仅预览 |

### 使用示例

假设你的文件夹结构如下：
```
./实验报告/
├── 张三-实验一.docx
├── 李四的实验报告.doc
```

#### 1. 预览操作系统实验1的重命名结果
```bash
python main.py ./实验报告 -s 操作系统 -e 1
```
输出预览：
```
目标目录: /path/to/实验报告
实验次数: 1
预览模式

[预览] 张三-实验一.docx -> 实验1_2026001_张三.docx
[预览] 蔡萍的实验报告.doc -> 实验1_2026002_李四.doc
...
==================================================
处理完成: 共 2 个文件
可重命名: 2 个, 跳过: 0 个
提示: 使用 --execute 参数实际执行重命名。
```

#### 2. 执行计算机组成原理实验2的重命名
```bash
python main.py ./实验报告 -s 计算机组成原理 -e 2 --execute
```
输出：
```
[重命名] 张三-实验一.docx -> 实验2 - 2026001 - 张三.docx
[重命名] 李四的实验报告.doc -> 实验2 - 2026002 - 李四.doc
...

未提交人员名单 (1/2):
  王五
```

## 高级功能

### 输出未交人员名单

程序在每次运行结束后会自动输出未提交报告的同学名单，无需额外参数。处理完成后会对比 `config.py` 中的映射表，找出文件名中未匹配到任何姓名的同学。

输出示例：
```
未提交人员名单 (3/30):
  王五
  赵六
  钱七
```
如果所有同学均已提交，则会显示 `所有同学均已提交报告。`。

> 该功能在预览模式和执行模式下均生效。

### 使用正则表达式匹配姓名

如果你的同学们提交文件名中的姓名形式多样（如 “张三” / “ZhangSan” / “三张”），可以在 `config.py` 的键中使用正则表达式，并修改 `main.py` 中一行代码。

**步骤：**
1. 在 `config.py` 中，将键写为正则表达式字符串（无需转义）：
   ```python
   OS_MAPPING = {
       r"(张三|ZhangSan)": "实验{exp_num}_2024001_张三",
       # ...
   }
   ```
2. 打开 `main.py`，找到 `match_name` 函数中的这行：
   ```python
   pattern = re.escape(key)
   ```
   将其修改为：
   ```python
   pattern = key
   ```

重新运行即可启用正则匹配。

### 支持其他文件类型

默认只处理 `.doc` 和 `.docx` 文件。如需支持 PDF、TXT 等，修改 `main.py` 顶部的 `WORD_EXTENSIONS` 元组即可：
```python
WORD_EXTENSIONS = ('.doc', '.docx', '.pdf', '.txt')
```

## 常见问题

**Q1：预览时显示“未匹配到任何姓名”怎么办？**  
A：请检查文件名中是否包含 `config.py` 中定义的姓名关键字（如 “张三”）。文件名中只要包含完整的姓名即可匹配，例如 `张三_实验报告.docx`、`实验1-张三.doc` 均可匹配。

**Q2：重命名后文件扩展名会丢失吗？**  
A：不会。程序会自动保留原文件的扩展名（如 `.docx`）。

**Q3：如果目标文件名已存在会怎样？**  
A：程序会检测冲突并跳过该文件，输出 `[冲突]` 提示，避免覆盖已有文件。

**Q4：如何增加新科目？**  
A：在 `config.py` 中添加新的映射字典，并将其加入 `SUBJECTS` 字典即可。也可以使用 `generate_config.py` 脚本重新生成。

**Q5：能否处理子文件夹中的文件？**  
A：当前版本仅处理指定文件夹根目录下的文件，不支持递归子目录。如有需要，可自行修改 `os.listdir` 为 `os.walk`。

**Q6：Windows 系统下能否使用？**  
A：完全支持。文件路径和重命名操作均兼容 Windows。
